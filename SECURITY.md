# Security Policy

## Supported Versions

Security fixes are applied to the latest release line.

## Reporting A Vulnerability

Please report vulnerabilities privately to the maintainers before public disclosure.

Include:

- affected version
- reproduction details
- expected impact

## Hardening Recommendations

- Use `strict-lock = true` in production.
- Use `enforce-trusted = true` with `trusted-packages`.
- Use private package indexes and explicit version pinning.
